Parksy Automation OS
1인 창작 공장 시스템 · Parksy Series
▶ YouTube

말만 해서 시스템까지 가고 싶다. I want to get from speech to a full working system.

처음 충격은 LLM이 코드까지 작성한다는 사실이었다.

The first shock was realizing an LLM can write full code on command.

텍스트 답변을 넘어서 파이썬, 웹코드, 아키텍처까지 만들어내는 걸 보고 욕망이 생겼다: “말 → 시스템.”

Beyond text, it produced Python, web code, and even architecture flows—triggering the desire for “speech → system.”

GitHub = 나의 출판사이자 코드 공장의 본체라는 인식이 핵심이었다. Realizing GitHub is both a publisher and the core of my code factory was the turning point.

모바일에서 모든 걸 처리하려 했으나 ffmpeg, Node, Docker 같은 공정은 PC가 필수였다.

I attempted to do everything on mobile, but processes like ffmpeg, Node, and Docker required a PC.

폰은 왕이지만, 공장은 PC에 있어야 한다. The phone is king, but the factory must live on the PC.

PWA로 인샷을 대체하려 했지만 Vercel의 실행 시간 제한과 폰 로컬 파일 접근 문제로 실패.

PWA couldn't replace InShot due to Vercel timeouts and inability to directly access phone storage.

핵심은 “누가 / 어디서 / 무엇을 / 어떻게 돌리냐”의 구조다. The system depends entirely on “Who / Where / What / How” the work runs.

폰은 명령·기획·확인, PC는 공장, GitHub는 출판사, PWA는 경량 자동화, APK는 인샷 대체.

Phone = command. PC = factory. GitHub = publisher. PWA = lightweight automation. APK = true InShot replacement.

ChatGPT는 두뇌, Claude는 공장장, Grok은 캠페인, Copilot은 편집부다. ChatGPT is the brain, Claude the factory chief, Grok the campaign unit, Copilot the editing team.

각각의 역할을 명확히 분리함으로써 박씨 1인이 40명 팀처럼 움직인다.

By separating roles, Parksy operates like a 40-person studio alone.

말 → GitOps → 빌드 → 출판 → 유통 Speech → GitOps → Build → Publish → Distribution

말하면 ChatGPT가 구조화하고, GitHub에 저장되면 Actions가 빌드하고, Pages/Vercel이 전 세계로 퍼트린다.

Speak → ChatGPT structures → GitHub stores → Actions build → Pages/Vercel distribute worldwide.

말하면 기록된다. If you speak, it is recorded.

기록되면 빌드된다. If it is recorded, it gets built.

빌드되면 퍼블리시된다. If it is built, it gets published.
