02 · Character Model

형근 캐릭터 모델 v2

"사람 한 명 = 하나의 세계관"

형근은 21살, 대학교 2학년 허브·약초 전공. 2년 뒤 군입대로 1.5년 공백기가 예정돼 있고, 그리스인 여자친구와 글로벌한 관계를 유지 중이다. ChatGPT를 곧잘 활용하고 태블릿도 쓰는, 디지털·모바일 친화형 청년이다.

Mobility 역마살 · 해외 동선 선호
Identity Split 아버지 부자/아니다 콤플렉스
Fast Learner ChatGPT·태블릿 활용 OK
Nature Craft 허브·향·자연물 감각
현실 vs 이상 간극 체감 부모 재정 인식 혼재 박씨 말에 공명하는 주파수 있음
Character Drivers

1) 이동성 (Mobility)
해외·여행·이동을 전제로 하는 삶. 정착형 공장보다, 이동형 콘텐츠·브랜드에 적합.

2) 인지 부조화 (Identity Split)
아버지의 부자/아님에 대한 콤플렉스, 현실과 눈높이 사이의 분리감. 브랜드·정체성 설계가 중요한 축.

3) 디지털 중간자 (Fast Learner)
ChatGPT·태블릿 사용에 거부감이 없고, 프롬프트만 잘 던져줘도 금방 따라오는 타입.

4) 물성 기반 스킬 (Nature Craft)
허브, 향, 채집, 건조, 우려내기 같은 손 감각은 그대로 글로벌 상품의 원천이 된다.

03 · Channel Goals

YouTube 채널의 목적

"실존 청년의 삶을 창업·교육 시스템으로 압축"

이 채널은 단순한 강의 채널이 아니다. “형근이라는 실존 인물의 삶을, 창업 시뮬레이션 콘텐츠로 전환하는 실험”이다. 박씨는 통화만 하고, 나머지는 시스템이 처리한다.

실존 로그 기반 강의 통화 → 강의 앱 자동화 선택지형 비즈니스 설계 10년짜리 생존 전략
Primary Goals

① 형근의 고민·환경·관계를 “교과서가 아닌, 생방 로그”로 보여준다.
② 그 로그를 구조화해서 PWA 강의 앱과 YouTube 콘텐츠로 만든다.
③ 시청자는 형근을 보면서 자신의 삶을 역으로 설계해본다.

What viewers actually get
  • AI 시대 청년이 어떻게 자기 전공·연애·가족·군대까지 포함해 인생을 구조화하는지 관찰.
  • “전공 + 이동성 + 연애 + 돈 콤플렉스” 같은 현실적인 변수들을 하나의 OS로 보는 시각.
  • 통화 한 번이 어떻게 강의, 앱, 비즈니스 옵션으로 변환되는지 전체 파이프라인.

= 결과적으로, 이 채널은 “AI × 청년 현실 × 창업 시뮬레이터”라는 새로운 포맷의 프로토타입이다.

04 · Content Production Loop

콘텐츠 생산 루프 — 말 한 마디 = 강의 한 페이지

"말 → 구조화 → 코드/액션 → 빌드 → 출판 → 루프"

형근 프로젝트의 진짜 가치는, 박씨가 그냥 떠든 통화가 자동으로 강의와 앱으로 변환된다는 점이다. 여기서 박씨의 역할은 “잘 떠드는 것” 뿐이다.

01
전화 통화

형근과 그냥 수다. 고민, 상황, 감정, 목표를 대화로 뽑는다.

02
STT & 로그화

통화를 STT로 변환하고, 핵심 문장과 드라이버를 추출.

03
구조화 & MDX

문제를 재구성해서 PWA·YouTube용 MDX/HTML 구조 생성.

04
PWA & 영상

강의 앱·슬라이드·Shorts까지 자동 조합해서 배포.

Section Layout per Call
  • OpeningFrame — 오늘 통화의 핵심 한 줄.
  • Problem Statement — 형근이 실제로 막힌 지점.
  • Environment & Drivers — 가족, 전공, 연애, 군대, 돈.
  • Business Options 3~5개 — 정답이 아니라 가능성 리스트.
  • Prompt Engine Link — 형근이 직접 쓸 수 있는 프롬프트 템플릿.
  • Action Checklist — 다음 통화 전까지 시도해볼 작은 실험.
05 · Business Simulation Tracks

형근 전용 비즈니스 시뮬레이션 트랙

"정답이 아니라, 옵션의 지도"

형근에게 주어지는 건 “강요된 진로”가 아니라 여러 개의 옵션이다. 각 옵션은 전공, 이동성, 연애, 가족, 돈 콤플렉스 등 현실 변수를 그대로 반영한 시뮬레이션 트랙이다.

Track A — 로컬 허브 브랜드 “산에서 온 허브” · 제품 기반
A
  • 허브차, 말림팩, 드립팩 등 원물 기반 상품 설계.
  • 형근이 실제로 다니는 산·장소를 스토리텔링에 포함.
  • 로컬 카페·플리마켓·공방과의 콜라보 가능.
  • 장점: 전공 활용, 군휴학 시기에도 준비 가능.
Track B — 콘텐츠 크리에이터 6초 ASMR · Short-form 기반
B
  • 허브를 자르고, 말리고, 물에 우려내는 장면을 6~10초 루프로 제작.
  • AI 음원(소라, 그록)을 입혀 감성 Loop 콘텐츠로 재가공.
  • 해외 시청자도 이해 가능한 비언어적 구조.
  • 장점: 이동성·역마살과 찰떡궁합.
Track C — 허브 PWA 사전 교육형 포트폴리오
C
  • 형근이 알고 있는 허브·약초를 사진+메모로 PWA에 누적.
  • 성분, 효능, 채집 위치 등을 간단히 기록.
  • 이게 곧 취업·해외 진학·콜라보 제안용 포트폴리오가 된다.
Track D — 이동성 라이프스타일 브랜드 연애·여행·향의 세계관
D
  • “형근이 세계 허브 찾기” — 여행과 허브가 합쳐진 서사.
  • 그리스 여자친구와의 감각적 경험을 향·차·영상으로 번역.
  • 청년층이 공감할 만한 현실적 연애+진로 서사 포함.
Why "Options", not "Orders"

이 프로젝트에서 박씨의 역할은 “인생 코치”라기보다 “옵션 맵을 그려주는 설계자”에 가깝다.

  • 지시가 아니라, 선택지.
  • 한 번 고정된 진로가 아니라, 여러 트랙의 시뮬레이션.
  • AI 시대에 필요한 것은 “진로 추천”이 아니라 “옵션 설계 능력”.

= 결국, 이 채널의 핵심 상품은 “옵션 사고방식”이다.

06 · System Architecture

형근 프로젝트 아키텍처 (GitHub 기준)

"GitHub 페이지 1장 = 한 통화의 압축 로그"

이 HTML 파일은 그대로 GitHub Pages에 올릴 수 있는 단일 페이지다. Parksy OS에서 보면, 형근 프로젝트는 다음과 같은 스택으로 굴러간다.

Layer 01

Conversation Capture

- 전화 통화 녹음
- STT 변환 (Whisper / etc.)
- 형근 OS 로그로 정리
Input Layer · 말 기반 데이터
Layer 02

MDX / HTML Generator

- ChatGPT가 로그를 구조화
- MDX 또는 HTML 템플릿에 주입
- 섹션별 컴포넌트 구성 (Intro / Tracks / Loop)
Structure Layer · 강의 구조
Layer 03

PWA & YouTube Binding

- PWA 강의 앱에서 이 페이지를 하나의 모듈로 사용
- YouTube 채널 설명/고정 댓글에서 링크 연결
- Shorts·강의 영상과 상호 참조 구조 설계
Delivery Layer · 출판 & 배포
GitHub Usage Idea

- repo: eae.kr 혹은 별도 hyunggeun-sim 레포
- path: /hyunggeun/index.html or /sim/hyunggeun.html
- 이후 Claude MCP / Copilot Agents에게 “이 파일을 템플릿으로 삼아, 통화마다 새로운 HTML 생성” 명령 가능.

07 · Philosophy

이 프로젝트가 진짜로 하려는 것

"Edu + Art + Engineer = 실존 한 명을 OS로 돌리기"
“사람 한 명 = 하나의 OS.”
형근의 전공, 연애, 가족, 군대, 돈 고민을 모두 포함해서 한 번에 돌려보는 창업·콘텐츠·교육 실험.

여기는 교과서가 아니라, 실존 로그가 출발점이다. EduArt Engineer 관점에서 보면, 형근 프로젝트는 다음 세 가지를 동시에 건드린다.

Edu · 실제 청년의 삶을 교재화
Art · 전공·연애·콤플렉스까지 서사화
Engineer · GitHub·PWA·YouTube로 시스템화
Parksy View
  • AI 시대의 진짜 자산은 “좋은 이력서”가 아니라 “잘 기록된 삶의 로그”다.
  • 창업은 거창한 비전보다, 현재 환경을 구조화하는 능력에서 시작된다.
  • 실존 인물 한 명을 제대로 돌릴 수 있다면, 그 패턴은 누구에게나 적용 가능하다.

형근 프로젝트는 “한 명의 팬”에서 시작해서, “한 세대의 사용 설명서”로 확장될 수 있는 실험이다.
