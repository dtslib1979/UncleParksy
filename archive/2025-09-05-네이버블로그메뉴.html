네이버블로그메뉴

시각 다이어그램 · 캡처→발행 파이프라인 (SVG)

JS 없이 SVG로 그린 흐름도

1) 전제 (박씨님 미러링)

  • 입력: 삼성 키보드 음성 입력 위주
  • 성격: 철학 · 드로잉 · 시스템 → 정리보다 흘려보내기
  • 디바이스: 폰(기록) · 태블릿(그림) · PC(백업)
  • 플랫폼 전략: 네이버=아이디어 창고 → Tistory/GitHub/YouTube 확장

2) 게으른 블로깅 루틴

  1. 폰에서 음성 입력으로 초안
  2. 템플릿 3종 중 하나 불러오기 (일상/리뷰/작업)
  3. 발행 직전 맞춤법 검사 → 제목 붙이고 발행
  4. 주 1회 PC에서 Tistory/GitHub로 정리·백업
“네이버 블로그는 내 작업장의 낙서판이다.”

3) 메뉴별 활용 매칭표

메뉴박씨님 맞춤 활용법 (편하게 쓰기)기대 효과
장소평소 OFF, 식당 리뷰 때만 자동 지도 첨부귀찮은 입력 최소화
내돈내산경험담/리뷰 시 버튼으로 인증신뢰도 확보
스티커안 써도 OK. 강조 컷 1장만 가볍게최소 편집으로 분위기
링크글 끝에 “깊이 보기 → Tistory/GitHub/YouTube”허브 전략
인용구내 철학 워딩 한 줄을 인용 블록으로핵심 문장 강조
구분선“이미지·표 → --- → 내 생각” 구간 나눔가독성↑
맞춤법 검사발행 직전 1회 클릭오타 최소화
템플릿일상/리뷰/작업 3개만 저장해 즉시 사용세팅 시간 0
음성 입력메인 입력 장치타이핑 스트레스↓
파일필요시 PDF/이미지 1~2개 첨부부가자료 제공
내 클립짧은 작업 클립으로 네이버 내 노출 확보최소 영상 운영

5) Q&A (아코디언)

왜 템플릿은 3개만?

대부분의 포스트는 일상/리뷰/작업 기록의 3패턴으로 수렴합니다. 옵션을 줄이면 선택 피로가 줄고 발행 속도가 빨라집니다.

내돈내산은 언제 쓰나?

“내가 직접 결제/방문한 후기”에만. 신뢰 배지가 CTR과 체류시간에 기여합니다.

영상(내 클립)은 꼭 필요?

짧은 클립 1개만으로도 네이버 내 노출을 끌어올릴 수 있습니다. 과하게 편집할 필요는 없습니다.
