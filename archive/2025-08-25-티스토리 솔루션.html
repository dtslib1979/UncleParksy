티스토리는 왜 상품적 가치가 낮은가?
  • 광고 단가 및 네트워크 효과 약함(SEO, network effects).
  • 확장성 낮음 → 실험실(lab)·메모장(notepad) 포지션이 적합.
  • 상품화(monetization)는 GitHub Pages, WordPress, Substack 등으로 전환.
“쓸모없는 삽질” 같았던 티스토리 스터디의 실익은?
  • 대화→문답 추출→아코디언→HTML→발행 공정(production line) 체화.
  • 플랫폼 한계·실패 데이터를 직접 검증 → 다음 무대로 넘어갈 근거.
  • 결론: 돈은 0점, 시스템 훈련은 90점.
흔한 “GPT→HTML→블로그”와의 차별점은?
  • 결과물이 아니라 엔진을 설계: 프롬프트 모델링(prompt modeling)·메타모델링(metamodeling).
  • 재사용 가능한 메타프롬프트(meta‑prompt)·템플릿·검증 규칙을 표준화.
글로벌 확장(i18n)은 구조적으로 가능한가?
  • 가능. 정적 배포(static hosting) + 버전관리(Git) + hreflang/canonical.
  • KR 원문 + EN 동시 발행, Substack/Medium로 영문화 페더레이션.
티스토리를 “메모장(notepad)”으로 쓰는 이유는?
  • 초안·중간 저장에 최적: 카테고리·검색·미리보기 지원.
  • 최종본은 GitHub Pages/WordPress로 승격(canonical 관리).
초안 툴 정렬: 삼성노트 vs 메모장 vs GitHub?
  • 삼성노트(Samsung Notes): 드로잉/판서(whiteboard) 전용.
  • 메모장(plain text/Markdown): 한글 초안(kor‑first drafting).
  • GitHub: 결과 저장소(archive & deploy), 정본 관리(single source of truth).
WebAppsBook Cast 패키지의 구성은?
  • 마더시드 백서 웹앱(whitepaper webapp).
  • 프롬프트 메타모델링 앱(prompt metamodeling app).
  • 메타프롬프트 표준(meta‑prompt standard).
  • 강의 영상(YouTube 판서, whiteboard) 동시 발행.
최소 기술 원칙은 무엇인가?
  • Zero‑JS(상호작용은 <details>/<summary>), 복사 기능만 JS.
  • 리스트 기반(text‑first), 접근성(a11y), 모바일 우선(responsive).
  • 전문용어 영문 병기(English terms) 의무.
3‑스텝 배포 루틴은?
  • 메모장 초안 → 티스토리 임시(메모장) → GitHub Pages 최종.
  • i18n 빌드/메타(OG, canonical) 추가 → Substack/Medium 교차 게시.
수익화 루트는?
  • 퍼널(funnel): Free 티저 → $9 미니 → $29 스탠다드 → $99 프로.
  • 상품: 프롬프트 팩(prompt pack), 메타모델 가이드, HTML/MD 템플릿.
