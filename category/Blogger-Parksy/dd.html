키워드 한 줄 요약 말 → 구조화 → 코드/액션 → 빌드 → 출판 → 루프 One-line summary: Speech → structuring → code/action → build → publishing → feedback loop.

출발점은 “말만 해서 시스템까지 가고 싶다”는 욕망이다. It all starts from the desire to “reach a full system using speech only.”

PC는 이제 ‘내가 만지는 컴퓨터’가 아니라, 클로드가 대신 굴리는 자동화 공장이다. The PC is no longer “my personal machine” but an automated factory driven by Claude.

폰은 왕이다. 명령 내리고, 결과를 보고, 배포하고, 소비한다. The phone is king: it issues commands, checks results, publishes, and consumes.

인샷 대체는 PWA가 아니라 APK가 본질에 더 가깝다. Replacing InShot is fundamentally closer to an APK than to a PWA.

백서/튜토리얼/세계관/웹툰/쇼츠 전부 “말 → 구조화 → 코드/액션 → 출판”으로 묶인다. Whitepapers, tutorials, worlds, webtoons, and shorts are all bound by “speech → structure → code/action → publishing.”

최종 3원칙: 말하면 기록되고, 기록되면 빌드되고, 빌드되면 퍼블리시된다. Core three principles: If you speak, it’s recorded; if recorded, it’s built; if built, it’s published.
